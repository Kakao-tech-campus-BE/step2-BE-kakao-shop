## **과제**
### 1. 요구사항 시나리오를 보고 부족해 보이는 기능을 하나 이상 체크하여 README에 내용을 작성하시오.
   
기본 서비스에 해당하는 상품 검색, 옵션 선택, 장바구니 등의 로직은 알맞게 들어가 있다고 생각한다.
하지만 유저관리 영역에 해당하는 로그아웃 및 회원탈퇴 기능이 구현 되어 있지 않아 이 부분이 추가 되면 좋겠다는 생각이 들었다. 또한 회원가입을 할 때 기입 했던 정보들을 수정할 수 있는 마이 페이지(프로필 페이지)를 만들어 CRUD 기능을 구현할 수 있게 만들면 좋을 것 같다.

**로그아웃** <br>
로그인을 한 상태일 때 로그아웃 버튼을 누르기 -> 로그아웃

**마이페이지 - 정보 수정, 회원 탈퇴** <br>
로그인 상태일 시에 본인의 개인정보를 수정할 수 있게 만든다.
로그인 상태에서 회원탈퇴 버튼 누르기 -> 비밀번호 한번 더 입력 -> 정말 탈퇴 하겠냐는 알림 창 -> 회원 탈퇴
<br>
<br>

### 3. 배포된 서버에 모든 API를 POSTMAN으로 요청해본 뒤 응답되는 데이터를 확인하고 부족한 데이터가 무엇인지 체크하여 README에 내용을 작성하시오.

배포된 서버에 모든 API를 테스트 해본 후, 어떤 데이터가 추가되면 좋을지 생각해 보았다.

- 전체 상품 목록 조회
    - 상품의 별점
    상세 페이지에 존재하지만 목록에도 정보가 나와 있으면 좋을 것 같다.
    - 정가, 할인율
    고객들이 상품을 선택하는 요소들 중 가장 크다.
- 장바구니 조회
    - 상품의 이미지
    상품의 이미지가 같이 나와 있을 경우 장바구니에 어떤 상품이 들었는지 더 수월하게 확인할 수 있을 것 같다.
- 결제하기 - (주문 인서트)
    - 받은 혜택
    총 할인받은 혜택을 알려주면서 본인의 소비가 합리적이라는 것을 상기시켜 준다. 이때 최종 가격 뿐만 아닌, 정가로 구매 했을 시의 가격까지 표시하면 소비 촉진에 도움이 될 것 같다.
- 주문 결과 확인
    - 상품의 이미지
    앞서 말한 바와 같이 이미지가 있으면 구입한 상품을 더 알아보기 쉬울 것 같다.
